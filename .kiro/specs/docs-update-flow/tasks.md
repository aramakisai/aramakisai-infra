# Tasks Document - ドキュメント更新フロー (docs-update-flow)

本タスク定義書は、要件定義書および基本設計書に基づいて策定された、具体的な実装ステップと各タスクの検証方法（アクセプタンス基準）を定義します。

---

## 🛠️ タスク一覧

### Task 1: 主要ドキュメントの再構成とダイエット（軽量化）
- **Task 1.1**: `CLAUDE.md` のリファクタリングとダイエット
  - **内容**:
    - 末尾の `Agentic SDLC and Spec-Driven Development` セクションを削除し、[GEMINI.md](../../GEMINI.md) へのリンクに一本化。
    - 詳細なインフラ技術構成や変数定義などを `.kiro/steering/tech.md` および `structure.md` へ移譲。
    - 一時的な課題（既知のギャップ等）や詳細な手順記述を削除・圧縮し、全体の行数を 300 行以下にスリム化。
  - **検証方法**: `CLAUDE.md` の行数が 300 行以下になっており、不要な重複が削除されていること。
- **Task 1.2**: `.kiro/steering/tech.md` の更新とダイエット
  - **内容**: `CLAUDE.md` から移譲されたプロバイダー詳細、変数一覧、詳細な設定フラグの解説などを簡潔に追記・統合。
  - **検証方法**: ファイルが 150 行以下に収まっており、インフラの技術的意思決定が網羅されていること。
- **Task 1.3**: `.kiro/steering/structure.md` の更新とメモリ同期ルールの追加
  - **内容**:
    - ディレクトリ構成やサービス追加パターンの記述を整理・圧縮。
    - 末尾に「Kiro仕様完了時のプロジェクトメモリ同期プロセス（転記基準、手順）」のルールを追記。
  - **検証方法**: ファイルが 150 行以下に収まっており、仕様完了時のドキュメント更新ルールが明記されていること。

### Task 2: 相対リンク検証スクリプトの作成
- **Task 2.1**: リンクチェック用の Python スクリプト `scripts/check-links.py` の実装
  - **内容**:
    - 標準ライブラリのみを使用し、Markdownファイル（`*.md`）から `[テキスト](パス)` および `(file:///パス)` 形式のリンクを抽出するスクリプトを作成。
    - 外部リンクやアンカーリンクを除外し、ローカル相対パスおよび `file:///` 形式のパスの存在確認を行う。
    - リンク切れが検出された場合は詳細を出力し、終了コード `1` で終了、正常時は `0` で終了。
  - **検証方法**: テスト用のダミーリンクを含む一時的な Markdown ファイルを作成し、スクリプトがリンク切れを検知して終了コード `1` を返し、正常なリンクのみの場合は `0` を返すことを確認する。

### Task 3: 変更時更新ナビゲーションフローとAIルールの適用
- **Task 3.1**: `CLAUDE.md` への「変更時更新ナビゲーション」の追加
  - **内容**: インフラ変更（Ansible, Terraform, ArgoCD等）時に、どのドキュメントをどのような基準で更新すべきかの簡易フロー/チェックリストを `CLAUDE.md` に追加する。
  - **検証方法**: `CLAUDE.md` に変更時更新ナビゲーションのセクションが正しく追加されていること。
- **Task 3.2**: `.kiro/steering/` への「AI用ドキュメント自律同期ルール」の適用
  - **内容**: `.kiro/steering/structure.md` 等の末尾に、AIエージェントがインフラ変更時に自律的にドキュメントを同期更新するためのルールを追記する。
  - **検証方法**: 指定の steering ドキュメントにルール文言が正しく追加されていること。

### Task 4: 動作検証と仕様完了
- **Task 4.1**: リンク検証スクリプトの実行とプロジェクト全体のリンククリーンアップ
  - **内容**:
    - `uv run scripts/check-links.py` を実行し、既存の Markdown ファイル内に壊れたリンクがないかを確認。
    - もしリンク切れがあれば修正する。
  - **検証方法**: `uv run scripts/check-links.py` が正常に終了（コード `0`）すること。
- **Task 4.2**: 仕様 (Spec) を完了し、メモリを同期
  - **内容**: `spec.json` を更新し、フェーズを `completed` にする。今回作成された新規スクリプトや同期ルールについて、プロジェクトメモリを同期する。
  - **検証方法**: 全タスクが完了し、`spec.json` の `phase` が `completed` になっていること。
